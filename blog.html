10/2/13

I’m currently developing a roguelike library in Scala. It’s currently quite usable, but there’s a few functions I need to implement before I can release it. If you want to try it already, it’s available on Bitbucket. You’ll also need my Scala-common library, which provides a bunch of utility functions and classes such as vectors, a configuration file reader, and the Mersenne Twister RNG. To use these libraries, run sbt package on the command line, go into target/scala-2.10, and copy the .jar to your project. To generate documentation, run sbt doc and open index.html in target/scala-2.10/api. When I release the library you’ll be able to download the jars directly and the documentation will be available online.

This console library differs from others in that it provides several varieties of console, such as hexagonal consoles, consoles that make it easy to display messages, and consoles that display tiles loaded from an image. Several of these consoles can be combined in one window.

The library is written in Scala, but it might also be usable in Java. However, it uses some Scala features such as partial functions that are hard to access from Java. If there’s demand for it, I might write a wrapper to make it easier to use from Java, but for now I’ll stick to Scala. In my opinion, Scala is significantly better than Java for most applications. For instance, you don’t have to declare the types of most variables (the compiler infers them), it supports operator overloading, and it’s great for functional programming. One other area where Scala is superior to Java is that it supports partial functions, which are functions written as a series of “case” statements. In this library, they allow you to handle input more concisely than you could in Java.

Here’s an example of a simple program using this library, which displays an @ sign that you can move around the screen (for people unfamiliar with roguelikes, the @ sign is typically used to represent your character). There are several other examples in src/Demo.scala.

import com.nathanstoddard._
import common._
import console._

object SimpleDemo {
  def main(args:Array[String]) {
    val window = new Window("Simple demo")
    val console = new TextConsole(window, IVec2(80,24))
    window.show()
    var youPos = IVec2(10,5)
    while (true) {
      console.clear()
      console(youPos) = '@'
      window.waitForKey {
        case Key.left  => youPos += IVec2(-1,  0)
        case Key.right => youPos += IVec2( 1,  0)
        case Key.up    => youPos += IVec2( 0, -1)
        case Key.down  => youPos += IVec2( 0,  1)
        case Key.escape => exit()
      }
    }
  }
}

The library is currently missing several features such as line of sight, field of view, partially transparent tiles, and integration with Swing. I’m not sure if all of them will be implemented before the first release.
